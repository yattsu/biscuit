# biscuit. — Custom E-Ink Firmware

## ⚠️ BUILD RULE — DO NOT COMPILE
Do NOT run `pio run`, `pio test`, `python -m platformio`, or any build commands.
Only read and edit source files. The owner builds manually.

Custom firmware for the **Xteink X4** e-ink reader, forked from CrossPoint Reader.
Adds offensive wireless testing tools, games, and utilities to an e-book reader platform.

## Hardware

| Spec | Value |
|------|-------|
| SoC | ESP32-C3 (single-core RISC-V, 160MHz) |
| RAM | 380KB SRAM (no PSRAM) |
| Flash | 16MB (currently ~94% used — budget is critical) |
| Display | 4.26" 800×480 e-ink (SSD1677), 1-bit monochrome |
| Input | 7 physical buttons: Back, Confirm, Up, Down, Left, Right, PageForward |
| WiFi | 2.4GHz 802.11 b/g/n |
| BLE | 5.0 (shared radio with WiFi — cannot use both simultaneously) |
| Storage | MicroSD card (FAT32, SPI bus shared with display) |

## Activity Architecture

Every screen is an `Activity` subclass. Required interface:

```cpp
class MyActivity final : public Activity {
  void onEnter() override;           // Initialize state, acquire radio, requestUpdate()
  void onExit() override;            // Release resources, shutdown radio
  void loop() override;              // Read buttons, update state, requestUpdate()
  void render(RenderLock&&) override; // clearScreen → draw → displayBuffer
  bool preventAutoSleep() override;  // Return true for long-running activities
  bool skipLoopDelay() override;     // Return true for time-sensitive activities
};
```

Key rules:
- `render()` runs on a separate FreeRTOS task — never block, never allocate large buffers
- Never call `render()` directly — use `requestUpdate()` to signal the render task
- `loop()` handles all input — never read buttons in `render()`
- Use `finish()` to pop activity, `onGoHome()` to return to home screen
- Radio: call `RADIO.ensureWifi()` or `RADIO.ensureBle()` in `onEnter()`, `RADIO.shutdown()` in `onExit()`

## Coding Conventions

- C++17, PlatformIO with Arduino framework for ESP32
- Prefer stack allocation over heap — 380KB total, every byte counts
- Use `snprintf` over `std::string` concatenation for formatting
- Use `static const` / `constexpr` for lookup tables (kept in flash, not RAM)
- No `std::map` or `std::unordered_map` in activities — too much flash overhead
- No `delay()` in `render()` — blocks the render FreeRTOS task
- Always handle Back button — user must always be able to exit
- i18n: use `tr(STR_KEY)` for user-facing text, add new keys to I18nKeys
- Hardcoded English is acceptable for tool-specific labels not worth translating
